# Audit Instructions

`audit_sample.csv` contains 40 conflict pairs sampled from the locked
synthetic benchmark (`scripts/sample_for_audit.py`, seed 2026, reproducible).
Every non-`human_*` column is generated by construction in code — it is
exactly what the benchmark generator produced, not a paraphrase or summary.

**No human annotation has been performed on this file.** The `human_*`
columns are empty templates. This protocol exists so an independent party
(or a future round of this project with real annotator access) can validate
the benchmark's construction; its presence here is not a claim that
validation happened.

## What to check, per row

1. **`human_is_extraction_correct`** (`yes`/`no`): Does `branch_a_value` and
   `branch_b_value` faithfully represent what a real user/agent would say for
   this `(entity, predicate)`? (For the MemoryAgentBench extension rows —
   see below — this checks the regex parser's extraction against the
   original context line, not the synthetic generator.)
2. **`human_is_conflict_type_correct`** (`yes`/`no`): Does
   `generated_conflict_type` (`orthogonal` / `resolvable` / `ambiguous`)
   match your own judgment of whether these two values genuinely conflict,
   and whether a confident answer is warranted?
3. **`human_preferred_resolution`**: If you think this conflict has a single
   correct answer, what is it? Compare against `generated_correct_value`.
4. **`human_should_flag`** (`yes`/`no`): Independent of the generator's
   label, would you flag this as unresolvable rather than pick a side?
5. **`human_notes`**: Free text — anything that seems off, ambiguous, or
   worth discussing.

## Extending to MemoryAgentBench content

`scripts/sample_for_audit.py` currently samples only the synthetic locked
run. To audit the MemoryAgentBench extension's regex-parsed triples instead,
see `results/final/parser_precision_self_audit.json` — that file already
contains the author's own self-audit (labeled as such, not independent) of
50 randomly sampled extractions against their source context lines. A
genuinely independent audit of that same sample would be the most direct
next step: the paper's Limitations section states plainly that whether
"resolvable" ground truth and the noise parameter reflect real-world
provenance reliability "remains completely unvalidated."

## What NOT to do

Do not fill in the `human_*` columns yourself and represent that as
independent validation in any paper draft — the whole point of this
protocol is that the annotator is someone other than the paper's author,
with no stake in the benchmark's results.
